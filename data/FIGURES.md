# Figures Guide — what each plot means and which numbers belong where

Critical distinction for the whole project: **idealized concept** vs **actual data**.
Keep them in separate sections of the report and never mix their numbers, or the
story contradicts itself (clean class definitions imply ~100% accuracy; the real
overlapping data is what gives the honest 85.7%).

---

## The two worlds (read this first)

| | Idealized / conceptual | Actual / training data |
|---|---|---|
| **What it is** | Clean Platonic definition of each class | The graded, overlapping data the KAN was trained & tested on |
| **Generated by** | `data/plot_archetypes_ideal.py` (hand-drawn canonical signals) | `data/generator.py` (continuous latent-factor model) |
| **Torque envelope violation** | hostile **87.6%**, others **0.000%** | hostile ≈**12%**, hobbyist ≈2%, benign ≈1% (graded, overlapping) |
| **Implied separability** | Perfect | Realistic overlap |
| **Use for** | "What each class *means*" (motivation) | "What the model actually faces" (data + results) |
| **NEVER** | quote its 87.6% / 0.000% as the training-data stats | call it the class "definition" |

The idealized 87.6% / 0.000% numbers come from the FIRST dataset, which was
abandoned because it produced a misleading 100% accuracy. They are valid only as
an illustration of the concept.

## Figure-by-figure

### 1. `archetypes_ideal.png`  — IDEALIZED concept
- **Section:** Concept / class definitions.
- **Shows:** one clean canonical track per class for all 5 parameters.
  - Altitude: benign steady ~100 m · hobbyist low/erratic ~45 m · hostile terrain-hug dive 95→20 m
  - Pitch: benign ~flat · hobbyist slight vibration · hostile aggressive ±30°
  - RPM: benign stable · hobbyist fluctuating · hostile high + peaks
  - Velocity: benign constant ~12 · hobbyist variable · hostile burst→loiter
  - Torque: benign ~55% · hobbyist ~70% (under) · hostile crosses 100% envelope
- **Caption:** "Idealized signatures of each intent class (conceptual; not training data)."
- **Numbers OK here:** the 87.6% / 0.000% idealized violation figures.

### 2. `sample_tracks.png`  — random real training tracks
- **Section:** Data (optional).
- **Shows:** one RANDOM track per class from the actual data. Overlapping/messy.
- **Caption:** "Representative random samples from the training data; note the
  realistic overlap between classes."
- **Numbers OK here:** none specific; illustrative only.

### 3. `separation_archetypes.png`  — medoid (most-representative real track)
- **Section:** Data (optional).
- **Shows:** the real track nearest each class mean. Still overlaps (the data is hard).
- **Caption:** "Most-representative real track per class; even archetypes overlap."

### 4. `separation_feature_space.png`  — PCA separation (KEY data figure)
- **Section:** Actual data / separability.
- **Shows:** all 6,000 tracks' 18 features projected to 2D. A benign→hobbyist→hostile
  gradient along PC1 (~61% variance = the aggression axis) WITH boundary overlap.
- **Caption:** "Class separation in feature space: clear ordering with realistic
  overlap — the overlap band is where misclassifications occur."
- **Why it matters:** visually justifies BOTH "the model works" and "85.7%, not 100%".

### 5. `model/training_report.png`  — RESULTS
- **Section:** Results.
- **Shows:** (left) train/val loss converging without divergence; (right) confusion
  matrix, test accuracy 0.857.
- **Caption:** "Training converges without overfitting; confusion is adjacent-class
  and symmetric."
- **Numbers OK here:** overall **85.7%**, easy 98.2%, hard 48.2% (from `model/metrics.json`).

---

## The one-sentence rule
Idealized figures + 87.6%/0.000% answer **"what is each class?"**;
actual-data figures + **85.7%** answer **"how well can we tell them apart in
realistic conditions?"** Keep those two questions — and their numbers — apart.
